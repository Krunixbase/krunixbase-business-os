# Moduł Kalendarz — Krunixbase Business OS

Moduł Kalendarz umożliwia zarządzanie wydarzeniami, zadaniami, spotkaniami, harmonogramami projektów oraz integracją z zewnętrznymi kalendarzami (Google Calendar, Outlook). Stanowi kluczowy element organizacyjny systemu Krunixbase Business OS.

---

## 1. Cele modułu kalendarza

- centralizacja wydarzeń i zadań,
- synchronizacja z kalendarzami zewnętrznymi,
- integracja z projektami i CRM,
- automatyzacja harmonogramów,
- powiadomienia mobilne,
- wizualizacja czasu pracy.

## 4. Workflow kalendarza

### **Przykładowy workflow operacyjny**
1. operator tworzy zadanie,  
2. zadanie pojawia się w kalendarzu,  
3. automatyzacja dodaje przypomnienie,  
4. operator wykonuje zadanie,  
5. kalendarz aktualizuje status.

### **Workflow projektowy**
1. etap projektu ma termin,  
2. etap pojawia się w kalendarzu,  
3. zadania projektowe synchronizują się,  
4. Gantt aktualizuje kalendarz,  
5. budżet aktualizuje koszty etapu.

### **Workflow CRM**
1. klient dodany do etapu „spotkanie”,  
2. automatyzacja tworzy wydarzenie,  
3. operator otrzymuje powiadomienie,  
4. spotkanie zapisane w historii klienta.

---

## 5. Automatyzacje kalendarza

Moduł kalendarza współpracuje z silnikiem automatyzacji IF→THEN.

### **Przykłady automatyzacji**
- IF zadanie utworzone → THEN dodaj do kalendarza,
- IF etap projektu przesunięty → THEN aktualizuj kalendarz,
- IF spotkanie za 1 dzień → THEN wyślij przypomnienie,
- IF klient zmieni status → THEN utwórz wydarzenie,
- IF faktura KSeF wymaga działania → THEN utwórz zadanie w kalendarzu.
